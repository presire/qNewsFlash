# Rules

ルールは、ファイルパスに基づいて自動的にロードされるドメイン固有の知識です。
現在編集しているファイルの種類に応じて、適切なルールが適用されます。

## ルール一覧

| ルールファイル | 適用パス | 内容 |
|----------------|----------|------|
| cpp-code.rules.md | `*.cpp`, `*.h`, `*.hpp` | C++/Qt コーディング規約 |
| cmake.rules.md | `CMakeLists.txt`, `*.cmake` | CMake ビルド規約 |
| json-config.rules.md | `*.json`, `*.json.in` | JSON 設定ファイル規約 |

## ルールの仕組み

### 自動ロード

Claude Code は、現在編集しているファイルのパスに基づいて、
一致するルールファイルを自動的にロードします。

例：
- `HtmlFetcher.cpp` を編集 → `cpp-code.rules.md` がロード
- `CMakeLists.txt` を編集 → `cmake.rules.md` がロード
- `qNewsFlash.json` を編集 → `json-config.rules.md` がロード

### パス指定

各ルールファイルの先頭で、適用するパスパターンを指定できます：

```
---
globs: ["*.cpp", "*.h", "*.hpp"]
---
```

## ルールとスキルの違い

| 特徴 | ルール | スキル |
|------|--------|--------|
| ロードタイミング | ファイル編集時に自動 | 明示的呼び出し |
| 目的 | ドメイン知識の提供 | ワークフロー実行 |
| スコープ | ファイルタイプ別 | タスク別 |

**使い分け:**
- 「このファイルタイプではどう書くべきか」→ ルール
- 「このタスクをどう実行するか」→ スキル

## ルールファイルの形式

```markdown
---
globs: ["*.pattern", "*.another"]
---

# ルール名

このルールの説明。

## 規約

### 命名規則
- 規則1
- 規則2

### パターン
```language
// コード例
```

## 禁止事項
- 禁止1
- 禁止2
```

## ルールの拡張

新しいルールを追加する場合：

1. `rules/` ディレクトリに新しい `.rules.md` ファイルを作成
2. ファイル先頭に `globs` を指定
3. ルール内容を記述

例：`rules/docs.rules.md`

```markdown
---
globs: ["docs/**/*.md", "*.md"]
---

# Documentation Rules

Markdown ドキュメントの書き方...
```
