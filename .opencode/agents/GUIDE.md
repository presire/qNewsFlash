# Subagents Guide

サブエージェントは、特定の専門領域に特化した自律的なAIアシスタントです。
独立したコンテキストウィンドウを持ち、複雑なタスクを委譲できます。

---

## 利用可能なサブエージェント

| サブエージェント | 専門領域 | 使用場面 |
|------------------|----------|----------|
| qt-cpp-developer | Qt5/Qt6 C++開発 | Qt Network、Signal/Slot、メモリ管理 |
| xpath-specialist | XPath/Webスクレイピング | ニュースサイト構造変更、XPath式作成 |
| cmake-builder | CMake/ビルド | 依存関係解決、クロスコンパイル |

---

## 使用方法

### タスク委譲

サブエージェントに複雑なタスクを委譲できます：

```
qt-cpp-developerサブエージェントに、HTTP通信クラスの実装を委譲してください。
```

```
xpath-specialistサブエージェントに、時事ドットコムの新しいXPath式の作成を依頼してください。
```

```
cmake-builderサブエージェントに、Qt6用のビルド設定を修正させてください。
```

### 自動選択

タスク内容に基づいて適切なサブエージェントが自動選択されます：

| トリガー語/タスク | 選択されるサブエージェント |
|-------------------|----------------------------|
| Qt Network API実装、Signal/Slot | qt-cpp-developer |
| エンコーディング変換、タイマー実装 | qt-cpp-developer |
| XPath式の作成・修正、HTML構造分析 | xpath-specialist |
| ニュースサイト追加、スクレイピング | xpath-specialist |
| CMakeビルドエラー、find_package | cmake-builder |
| クロスコンパイル、インストール設定 | cmake-builder |

## 各サブエージェントの詳細

### qt-cpp-developer

Qt5/Qt6 C++開発を専門とするエージェント。

**得意なタスク:**
- QNetworkAccessManager (HTTP/HTTPS通信)
- QString / QTextCodec (文字列処理、エンコーディング)
- QTimer (定期実行)
- Smart Pointers (QSharedPointer, QScopedPointer)
- Signal/Slot パターン
- Qt5/Qt6 互換性対応
- ネットワークリトライ処理
- メモリリーク防止

**タスク受領時の確認項目:**
- 目的の明確化
- Qt バージョン要件
- ネットワーク要件
- エンコーディング要件

### xpath-specialist

XPath クエリと Web スクレイピングを専門とするエージェント。

**得意なタスク:**
- XPath 1.0 式の作成・最適化
- libxml2 API の使用
- HTML/XML パース処理
- ニュースサイト構造分析
- Webサイト構造変更対応
- エンコーディング問題解決

**分析フロー:**
1. HTML 取得
2. 構造分析
3. XPath 作成
4. 検証

### cmake-builder

CMake ビルドシステムを専門とするエージェント。

**得意なタスク:**
- CMakeLists.txt の作成・修正
- find_package (Qt, libxml2, OpenSSL)
- ビルドオプション設定
- クロスコンパイル設定
- インストール設定
- CI/CD 連携

**診断フロー:**
1. エラー分類
2. 原因特定
3. 解決策提示

## ワークフロー例

### 1. 新しいニュースソース追加フロー

```
1. xpath-specialist
   - 対象サイトの構造調査
   - XPath 式の作成
   - テストと検証

2. qt-cpp-developer
   - 新しいクラスの実装
   - HTTP 通信の実装
   - エラーハンドリング

3. cmake-builder
   - CMakeLists.txt の更新
   - ビルド確認
```

### 2. ビルド問題解決フロー

```
1. cmake-builder
   - エラーメッセージの分析
   - 原因の特定
   - 解決策の提示

2. cmake-builder
   - 設定の修正
   - ビルドの再実行
   - 成功確認
```

### 3. Webサイト構造変更対応フロー

```
1. xpath-specialist
   - 変更の検知と分析
   - 新しい XPath 式の作成

2. xpath-specialist
   - テスト HTML での検証
   - 設定ファイルの更新
   - 動作確認
```

## サブエージェントとスキルの違い

| 特徴 | スキル | サブエージェント |
|------|--------|------------------|
| コンテキスト | 共有 | 独立 |
| 実行形態 | ワークフロー | 自律的エージェント |
| 呼び出し | 明示的/自動 | タスク委譲 |
| 用途 | 手順・知識の提供 | 複雑なタスクの委譲 |
| 応答速度 | 即座 | 処理時間あり |
| 対話 | 不可 | 可能 |

**使い分け:**
- パターンやベストプラクティスを知りたい → スキル
- 複雑な実装や調査を任せたい → サブエージェント

以下の場合は Oracle エージェントへの相談を検討してください：

|------|------|
